v1.3.8 (2025-10-21) - CLAUDE CODE PARITY RELEASE
-------------------------------------------------
NEW FEATURES:
✓ Full file operation suite matching Claude Code/Cursor:
  - read_file() with line numbers (cat -n format)
  - write_file() for direct file creation
  - edit_file() for surgical find/replace edits
  - glob_search() for fast file pattern matching
  - grep_search() for content search with regex
  - batch_edit_files() for multi-file refactoring

✓ Auto code block extraction and file creation:
  - Automatically extracts code from LLM responses
  - Creates files when user requests (e.g., "create fixed_version.py")
  - Supports Python, R, SQL, Bash code blocks

✓ Enhanced command interception:
  - cat/head/tail → read_file() with line numbers
  - echo > file → write_file()
  - sed -i → edit_file()
  - grep -r > file → grep_search() + write_file()
  - find → glob_search()

✓ Multi-step workflow support:
  - "Read file, find bugs, create fixed version" works end-to-end
  - "Search for pattern, write report to file" executes completely
  - Context-aware command chaining

IMPROVEMENTS:
✓ Planner prompt updated with file operation examples
✓ Interceptor ordering fixed (grep > file before plain grep)
✓ Better error handling with detailed debug messages
✓ Large file support with pagination (2000+ lines)
✓ Edge case handling (empty files, missing files, invalid regex)

BUG FIXES:
✓ Fixed grep_search() return value handling
✓ Fixed command interceptor conflicts
✓ Fixed code block extraction regex
✓ Fixed file path resolution for relative paths

VALIDATION:
✓ 10/10 score on comprehensive validation tests
✓ All critical workflows verified:
  - Read file with line numbers ✓
  - Identify bugs with line numbers ✓
  - Create fixed code files ✓
  - Generate bug reports (grep > file) ✓
  - Multi-step workflows ✓
